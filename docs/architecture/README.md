# Architecture Documentation

This directory is the home for documents that explain LabAssistant's system
boundaries, components, runtime structure, data flow, and integration contracts.
Architecture documents should describe current or approved direction, not
speculative designs.

Current architecture references:

- [`capabilities.md`](capabilities.md) — stable, transport-independent
  scientific workflow contracts and the capability backlog.
- [`api-readiness.md`](api-readiness.md) — external contract classifications,
  stable seven-read surface, hardening gate, and selected local-transport
  go/no-go.
- [`../ARCHITECTURE.md`](../ARCHITECTURE.md) — target module boundaries and
  migration guidance.
- [`../STANDALONE_APP.md`](../STANDALONE_APP.md) — standalone application and
  agent-access boundaries.
- [`../OPENLAB_ARCHITECTURE.md`](../OPENLAB_ARCHITECTURE.md) — OpenLab
  chromatography support architecture.
- [`../CONTEXT_ENGINE.md`](../CONTEXT_ENGINE.md) — context engine design.
