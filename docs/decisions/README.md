# Decision Records

This directory stores concise records of significant architectural and
engineering decisions. Add a numbered record when a change establishes or
materially alters module boundaries, runtime behavior, dependency policy, API
contracts, or development standards.

Each record should state the status, date, context, decision, consequences, and
related prompt. Do not create records for routine fixes or speculative ideas.

Decision records use filenames such as `001-short-decision.md`.
