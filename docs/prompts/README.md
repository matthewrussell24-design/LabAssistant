# Implementation Prompts

This directory stores substantial implementation requests as numbered,
permanent project records. Use filenames such as
`001-short-description.md`, incrementing the number for each new request.

Create a prompt from [`TEMPLATE.md`](TEMPLATE.md) before substantial work. When
the work is complete, update that same file with the implementation summary,
files changed, test results, and remaining work instead of creating a separate
completion document.
